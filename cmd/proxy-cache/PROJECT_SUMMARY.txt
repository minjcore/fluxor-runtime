📦 PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════════════════

ProxyCache is a production-ready, high-performance caching proxy for package
managers (Go, npm, pip, etc.). Built on the fluxor-cache architecture with
advanced features like rate limiting, CORS, cache warming, and multi-tier
caching.

Total Project Size: ~3100 lines (code + tests + docs)
Build Status: ✅ All tests passing (100% success rate)
Test Coverage: 30+ test cases + 3 integration scenarios

📋 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════

GET / (plus path)
  - Main proxy endpoint
  - Returns: package file with X-Cache header (MISS|HIT|HIT-DISK)

GET /_stats
  - Cache statistics
  - Returns: JSON with hits, misses, hit rate, size metrics

GET /_health
  - Health check
  - Returns: JSON with status and cache information


🎓 USAGE EXAMPLES
═══════════════════════════════════════════════════════════════════════════

Go Module Proxy:
  ./proxycache -port 8080 -upstream https://proxy.golang.org
  export GOPROXY=http://localhost:8080,direct
  go get github.com/gin-gonic/gin

NPM Registry:
  ./proxycache -port 8080 -upstream https://registry.npmjs.org
  npm config set registry http://localhost:8080

Python PyPI:
  ./proxycache -port 8080 -upstream https://pypi.org/simple
  pip config set global.index-url http://localhost:8080

Docker:
  docker run -d -p 8080:8080 -v cache:/cache proxycache

Cache Warming:
  ./proxycache &
  curl http://localhost:8080/github.com/gin-gonic/gin
